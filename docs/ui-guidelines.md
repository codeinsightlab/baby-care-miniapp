# 母婴小程序 UI 风格规范

本规范冻结一期验收前的小程序统一视觉方向，适用于 `baby-care-miniapp/pages/**`、`components/**`、`pages.json` 与 `static/**` 中的小程序用户界面。本轮规范只约束 UI 呈现，不改变业务逻辑、接口、数据结构或状态流。

## 1. 整体方向

- 关键词：温柔、干净、轻奶油风、卡片式、低压力照护感。
- 产品气质：像照顾宝宝的日常工具，不像后台管理系统，也不像医疗系统。
- 页面主语义围绕“宝宝”“照顾人”“照护”“提醒”“记录”，不使用“家庭 / 家庭成员 / 加入家庭 / 创建家庭 / 家庭管理”等用户可见概念。
- 页面首屏应清楚表达当前宝宝、当前状态或当前任务，避免列表和按钮堆叠成后台表格感。

## 2. 颜色

- 页面背景：浅奶油色 `#FFF8EE`，可少量使用 `#FFF7E8`。
- 主色：暖橙 `#F6B84B`，强调色可用奶油黄 `#FFD166`。
- 文字主色：`#2F2F2F`。
- 次级文字：`#7A7A7A`。
- 弱提示：`#A8A8A8`。
- 分割线：`#F0E6D6`。
- 喂奶：暖黄底 `#FFF3CE`，文字 `#D89C00`。
- 睡眠：浅紫底 `#F2E7FF`，文字 `#8D6BD1`。
- 尿布 / 护理：浅蓝或浅绿底 `#E8F7EC` / `#EDF6FF`，文字 `#62A66D` / `#6A8CCF`。
- 提醒：暖橙浅底 `#FFF1DF`，文字 `#D58B4D`。

## 3. 卡片

- 主内容统一使用白色卡片。
- 圆角统一 `16px - 20px`，小标签使用胶囊圆角。
- 卡片内边距统一 `16px` 左右，页面内卡片间距 `12px - 16px`。
- 阴影保持轻柔，不使用重投影。
- 不使用大面积纯灰背景。

## 4. 按钮与标签

- 主按钮：暖橙色胶囊按钮，文字白色。
- 次按钮：浅奶油底或暖橙浅底，文字暖橙 / 次级文字。
- 状态按钮与状态标签：小胶囊、浅底色、低饱和。
- 不使用后台系统式小方按钮。
- 禁用态只降低透明度，不改变布局尺寸。

## 5. 列表与时间轴

- 列表项使用柔和分割线 `#F0E6D6`。
- 时间、类型、备注应有清晰层级：时间较弱，类型加重，备注为次级文字。
- 状态标签靠右，使用圆角浅底。
- 历史记录、今日时间轴、提醒列表应共享同一类视觉节奏，避免表格感。

## 6. 空状态

- 空状态必须有温和文案，不只显示“暂无数据”。
- 推荐文案：
  - “今天还没有记录”
  - “暂无待执行提醒”
  - “还没有照顾人加入”
  - “还没有宝宝档案”
- 空状态图标使用圆润、低饱和、与 tabbar 一致的视觉语言。

## 7. 图标

- 底部 tabbar 必须有图标和文字：今日、记录、提醒、宝宝。
- 页面入口图标、空状态图标和 tabbar 图标风格统一：圆润、简洁、低饱和。
- 如缺少正式视觉资产，一期 MVP 可使用项目内轻量 SVG、CSS 圆形字标或 emoji / unicode 方案。
- 不使用外部版权不明素材，不提交字体文件。

## 8. 页面要求

- Splash / 登录：保持温和确认和加载语气，不暴露后台或调试感。
- 今日：突出当前宝宝和今日照护状态，记录卡片像照护概览，提醒避免长表格感。
- 记录：快速记录、历史记录、语音备注入口视觉统一，语音入口不呈现为调试功能。
- 语音备注 / 记录确认：强调“备注草稿、确认保存”，不做 AI 或语义解析暗示。
- 计划 / 提醒：表达为“个人提醒 / 照护计划”，提醒卡片不做后台列表。
- 宝宝 / 宝宝协作 / 宝宝创建 / 宝宝信息：围绕“宝宝”和“照顾人”，协作用“邀请照顾人 / 一起照顾宝宝”。

## 9. 本轮边界

- 只做 UI 风格统一。
- 不改后端、不改接口、不改数据结构、不改 service / api 数据语义、不改业务状态流。
- 不新增复杂组件库，不引入重型 UI 框架。
- 不为了视觉重构业务逻辑。
