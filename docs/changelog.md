# Changelog

## 2026-05-15

- 今日页“当前待执行”做最后一轮视觉语义修正：在 swiper 主卡顶部右侧恢复轻量 `待执行` 任务状态文案，避免主卡只剩类型后更像普通记录。

- 今日页“当前待执行”swiper 主卡右上角增加轻量分页点：2 条及以上待执行时显示，1 条时隐藏，点数量跟随当前 swiper 展示的待执行卡片数量。
- 分页点通过 `pendingIndicatorDots` 生成，当前 `activePendingIndex` 对应点使用暖橙胶囊高亮；swiper 滑动、点击上一项 / 下一项预览、完成 / 稍后后刷新列表均继续沿用原有索引同步与重置逻辑。
- 冻结并实施今日页“当前待执行”最终交互：改为 uni-app / 微信小程序内置 `swiper` 主卡切换 + 轻量上一项 / 下一项预览，不再继续 Wallet 手写 touch 动画方向。
- swiper 方案保留主卡完整操作、左右滑动切换、点击预览切换和 `activePendingIndex` 同步；移除横向 `scroll-view`、transform / z-index 重叠卡片和多层露出效果。
- 执行今日页“当前待执行”轻堆叠点击切换收敛版：移除上一版大高度、transform、z-index 多层重叠效果，改为 1 张主任务卡 + 最多 2 条后续预览条。
- 轻堆叠版保留 `activePendingIndex` 和点击预览切换；预览条仅展示时间和类型 / 标题，不展示备注、状态、完成 / 稍后按钮，避免像后台列表或一叠表单。
- 返工今日页“当前待执行”展示方案：上一版横向 `scroll-view` 卡片组不符合目标交互，本版改为 iOS 钱包式纵向堆叠卡片，默认第一张展开，后方卡片露出顶部并可点击切换到前面。
- 钱包式堆叠继续只展示前 5 条未完成 / 未取消提醒，完成 / 稍后按钮仅出现在主卡，按钮点击阻止卡片切换误触；本轮不修改后端、接口协议、登录链路、reminder 列表页、tabbar 或数据结构。
- 今日页“当前待执行”改为 iOS 风格横向滚动卡片：使用 `scroll-view scroll-x` 展示前 5 条待执行提醒，一屏突出主卡并露出后续卡片，增强首页任务焦点感。
- 今日页待执行卡片保留提醒类型、时间、备注、状态，并复用既有完成 / 稍后提醒操作；本轮不修改后端、接口协议、登录链路、reminder 列表页、tabbar 或数据结构。

- 优化 today / record / reminder / baby 主页面视觉层级：强化首页当前宝宝卡片、今日统计、当前待执行、今日时间轴、快速记录入口、语音备注入口、提醒操作按钮和当前宝宝标签。

## 2026-05-14

- 收口小程序无感登录主链路：所有环境都走无感登录，dev/test 通过 `dev-login` 换 token，prod 通过 `wx.login + mini-login` 换 token，登录通道只由 `config/env.js` 门禁决定。
- 调整请求错误处理：`401` 清 token / user 后按同一无感登录链路重试一次；`5xx`、网络异常、超时不清 token、不清 `BC_CURRENT_BABY_ID`，不强跳 login / splash / baby。
- 收口当前宝宝状态：`BC_CURRENT_BABY_ID` 仅在明确无权限 / 不存在或用户主动切换时清理，服务异常、后端重启、today / baby list 接口失败不再作为清理依据。
- 调整 Splash 启动边界：只做短暂登录态和当前宝宝入口恢复，不无条件拉 today summary 或今日提醒等首页业务接口。
- 增强 today / record / reminder / baby 页面错误态：业务请求失败只影响当前页面内容，展示错误态和重试入口，避免服务恢复后 tabbar 被回跳链路锁死。
- 冻结一期验收前小程序统一 UI 风格规范：轻奶油背景、白色卡片、暖橙主色、圆润图标、温和空状态、低压力照护感。
- 统一小程序页面视觉方向，覆盖 Splash、登录、今日、记录、语音备注、记录确认、计划、提醒、宝宝、宝宝协作、宝宝创建与宝宝信息页。
- 为底部 tabbar 增加项目内轻量 SVG 图标资源，避免外部素材授权风险。
- 本轮为 UI 风格统一，不涉及业务逻辑、接口、数据结构、后端或状态流变更。
